# Stellar Princesses Completion Goal

```text
/goal Complete the full product vision defined in PRD.md for Stellar Princesses.

Treat PRD.md as the authoritative product specification. Read AGENTS.md, PRD.md, GAME_DESIGN_DOC.md, TECHNICAL_DOC.md, the current source, and all relevant project skills before beginning. For any game-code work, load phaser-gamedev first and follow the Phaser 4.2 requirements.

Work autonomously until the PRD’s full-vision acceptance criteria are genuinely satisfied. Existing code and architecture are not constraints: retain, refactor, replace, or fully rewrite them when justified by product quality. Do not reduce the PRD to the limitations of the current prototype.

Primary requirements:

- Deliver a complete, polished, playable campaign—not a feature demonstration.
- Implement Lyra’s full narrative arc, all core companions, their personal quests, relationships, evolutions, and outcomes.
- Build the complete world, changing Nova Prime home base, distinct regions, exploration, dungeons, quests, bosses, final sequence, endings, and epilogues.
- Deliver strategically complete combat, progression, equipment, economy, saving, interface, controls, onboarding, difficulty, and accessibility.
- Follow the fixed art direction: premium high-resolution 2D pixel art using classic Final Fantasy IV-era visual grammar remastered to 2026 standards. Preserve crisp authored pixels, expressive portraits, rich environments, improved animation, modern effects, widescreen composition, and original Stellar Princesses identity. Do not ship prototype primitives, blurred upscaling, conventional 3D gameplay, or copied reference assets.
- Ensure audio, animation, VFX, writing, and presentation meet the PRD’s quality requirements.
- Keep the base game fully completable offline and free of live-service mechanics.

Execution rules:

1. Audit the current product against every PRD requirement and acceptance criterion.
2. Create and maintain a traceable implementation plan organized into playable vertical slices and completion milestones.
3. Resolve open questions through coherent product decisions, representative prototypes, testing, and player-experience reasoning. Record decisions without narrowing future possibilities unnecessarily.
4. Prioritize the complete campaign, core cast, core gameplay, regions, bosses, home-base consequences, and endings before optional breadth.
5. Implement in small, reviewable increments.
6. After each increment, run relevant automated tests, production builds, and browser-based gameplay validation. Visually inspect representative screens and correct regressions.
7. Maintain save compatibility when practical; add safe migration when product changes require it.
8. Fix defects encountered within the active product scope rather than documenting them indefinitely.
9. Update product and technical documentation when decisions or behavior change.
10. Commit each verified increment using the project’s sequential version convention, beginning after the latest existing version. Never commit broken or partially validated states.
11. Preserve unrelated user changes and never use destructive Git operations.
12. Continue until every PRD acceptance criterion is either verified complete or blocked by a dependency that cannot be resolved locally.

Definition of done:

- The complete campaign is playable from new game through meaningful ending and epilogue.
- Every core companion and required region is fully realized.
- Combat, progression, exploration, quests, home-base development, presentation, accessibility, controls, and saving meet the PRD.
- Final-quality visual content replaces prototype presentation throughout required gameplay.
- Production builds succeed.
- Automated, integration, visual, accessibility, and full-playthrough validation pass.
- No known progression blocker, save-loss defect, soft lock, placeholder required content, or critical visual inconsistency remains.
- A final PRD compliance audit provides evidence for every acceptance criterion.

Do not stop after planning, scaffolding, a vertical slice, or an MVP. Continue executing until the full PRD is complete.
```
